Installing Ubuntu

Written by Ethan Cheng on 2016-04-09

Modified by PChan on 2016-12-08

To install Ubuntu, you must first choose a version of Ubuntu to install. You can find downloads for recent versions here.

LTS means Long Term Support. This means that packages (ie security patches, kernel patches, and other user programs) in the ubuntu apt repositories will continue to be supported and updated for 5 years. The current LTS version is 16.04.1.

Non-LTS versions of Ubuntu are often less stable and are used as a pseudo-beta test for the upcoming LTS version. The next Non-LTS version, 17.04, is due to be released in April 2017.

For the remainder of this guide, we will be focusing on installing Ubuntu 16.04.1. The installation process for other versions and distributions are very similar.

Setting up your computer for the installation process

Depending on what type of computer you have, you will have a different process to set up your computer for installing Ubuntu. The first step to set up your computer is to obviously back up your files. SERIOUSLY. Back. Up. Your. Files. Not only is it just good practice to back up your files, but if something goes wrong in the installation process, you may lose your files forever. But don't worry! Don't be scared! Former Senpai Yicheng is a Linux Guru, and he's deleted his own filesystem on multiple occasions. It happens to everyone. Just store your files somewhere. :+1:

Useful tip: Your handy-dandy stuy.edu email account gives you unlimited Google Drive space. :+1:

The first step is to set up your computer to boot from a USB drive!

:computer: Windows
Disabling Fast Boot

If you have a custom-built computer, you can probably skip this step, unless your motherboard supports Windows 8+ BIOS control.

Starting with Windows 8, Microsoft has implemented BIOS settings to allow Windows to start up faster. This setting is called Fast Boot. What it actually does is it stores file that records certain states and settings on your computer when it shuts down, then only stops processes absolutely need to be shutdown. In essence, if Fast Boot is on, then your computer does not actually shut down completely.

To disable Fast Boot, open Control Panel and go into the Power Settings. Select Change Advanced Settings, and disable Fast Boot.

A tutorial for this prerequisite can be found here!.

Disabling Secure Boot

This step might not be necessary for everyone, but if you skipped it and a future step does not work, you should perform this step.

The next step to do on a Windows 8+ computer is to disable Secure Boot in your BIOS settings. Shutdown your computer, spam your BIOS setup key (while it is off. Yes, it sounds silly, but it makes it easier.) Turn on your computer while continuing to rapidly press your BIOS setup key.

The BIOS setup key varies from computer to computer. On most computers, such as Dell, HP, and Acer, the BIOS setup key is <F2>. On Asus computers, it is either <Esc> or <F2>, I forgot which one. If you don't know your BIOS setup key, look it up on Google.

If you are unable to get into your BIOS setup, there is a hidden trick on Windows to get into it. On the login screen, hold <Shift>, and click Restart. This should bring you to a screen that looks like this:

Select "UEFI Firmware Setings" to get to the Setup screen.

Once you get into your BIOS setup, it should look something like this:

Or like this:

Almost all computers will have BIOS screens that differ in some way. Because of this, I can't guide you completely through this step, but usually you want to go to the Boot or Security section using the arrow keys. Find the Secure Boot option and disable it. Usually, you need to hit Enter or Space to change the option.

After you have disable Safe boot or Secure Boot option, arrow over to the Exit tab and select Save Options and Exit.

:computer: Mac OSX

First things first: You're going to have to install certain tools to be able to boot into USB drives. I've only done this on a 2011 MacBook Air on OSX Yosemite. I've heard that there were issues with El Capitan interfering with things, so I can only point you in a general direction.

The tool you will be installing is called rEFInd. For El Capitan users, the creator of rEFInd loves you: Click me! :heart:

To install rEFInd, you must first download it at this website.

Now you'll have to delve into Terminal. Open Terminal.app. My personal terminal will look vastly different from yours, as I've done a ton of customization, but the commands you'll be issueing are the same.

Use cd to navigate to your Downloads folder:

You'll see that I have an earlier version of refind installed. I'll be guiding you through installing the new one (since I need to update too :smile:)

First, we'll create a special place to store rEFInd, as we'll be needing it forever and ever.

$ mkdir ~/refind/
$ unzip refind-bin-0.10.2.zip -d ~/refind

If we go into our new folder, you'll see refind has been installed.

$ cd ~/refind/refind-bin-0.10.2

We can execute refind-install by doing ./refind-install:

The command we want to use to install everything is:

$ sudo ./refind-install

You can use the --alldrivers flag if you want to. I've done it in the past and nothing bad happened to me, but take that with a grain of salt.

Reboot, and now you're all set to run boot from a USB!

Creating the USB

There are a multitude of ways to do this, but a way that is universal to all operating systems is to use the UNetBootIn tool.

You can install UNetBootIn here.

UNetBootIn is a fairly intuitive tool. If you have issues with it, Google, StackOverflow, CS Dojo, and the CS Dojo IRC are welcome to listen to your questions.

:warning: WARNING :warning:

The rest of this guide is a simple INSTALLATION guide. This means the rest of this guide will be assuming you want to erase your computer's hard disk completely and put Linux on it. If you wish to retain your original operating system, refer to the guide on Dual Booting.

Again: BACK UP ALL OF YOUR FILES. The installation process will WIPE your hard drive. If you don't know what wipe means, it means erase or reset to a blank state. So please. Back up your files.

DISCLAIMER: After this point, you've been warned enough. CS Dojo will not be held responsible for any lost data or important files.

Booting into the USB

Once you've set up your computer and backed up your personal files, reboot your computer with the USB plugged in. For :computer: Windows users, spam the boot menu button. This, like the setup button, varies from computer to computer. Google the key you need. The common ones are <F10>, <Delete>, and <F2>.

Once you've gotten to a boot menu, you'll see something like this:

Select the USB device that you want, and you should be all ready to go to boot.

For :computer: Mac users, rEFInd should have booted, and you'll see something like this:

.

Arrow over to your USB (it should be fairly obvious which one it is) and hit Enter to boot into it.

Starting the Installation

You should be booting into a screen that asks you if you want to try without installing, or install Ubuntu. Arrow over to the Install Ubuntu option and you should reach a screen that looks like this:

You should check both boxes, and hit Continue.

Next, you should reach something that looks like this:

Again, this is a guide for installing Ubuntu. NOT dual booting. As such, we will select the Replace option. If the Replace option is not there, select the Wipe Entire Drive and Install Ubuntu option.

Hit Continue.

The next couple steps are intuitive and ask you to set your time zone, username, password, etc.

After this, you should be done!
